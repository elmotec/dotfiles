# General Principles

- Be accurate and concise.
- Ask focused clarifying questions when requirements are ambiguous.
- If uncertain, say "I don't know" instead of guessing.
- Follow red-green TDD where practical.

## Scope And Files

- Do not create new `.md` files unless explicitly requested.
- Do not add throwaway example files unless requested.
- Follow repository guidance in `README.md` and `CONTRIBUTING.md`.

## Style And Quality

- Follow red-green TDD for development.
- Keep naming descriptive and consistent with the project.
- Prefer simple, readable solutions over clever ones.
- Include necessary validation and error handling.
- Avoid deprecated APIs and insecure patterns.

## References

- Read language-specific instructions when applicable:
  - `comments.instructions.md` for non-doc code files
  - `python.instructions.md` for Python files
  - `cpp.instructions.md` for C++ files

## Non-Negotiables

- Do not leave TODO placeholders or incomplete stubs in final code.
- Keep outputs deterministic and testable.
